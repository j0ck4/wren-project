# Wren — Руководство пользователя

Wren — нативный GTK4 / libadwaita клиент для управления
туннелями [WireGuard](https://www.wireguard.com/) на Linux.

🇬🇧 English version — [USAGE.md](./USAGE.md).

---

## Установка

### Шаг 1 — установите WireGuard tools

Wren — это GUI поверх `wg-quick`. Сам `wg-quick` должен быть
на хосте:

| Дистрибутив | Команда |
|-------------|---------|
| Ubuntu / Debian / Mint | `sudo apt install wireguard-tools` |
| Fedora | `sudo dnf install wireguard-tools` |
| Arch / Manjaro | `sudo pacman -S wireguard-tools` |
| openSUSE | `sudo zypper install wireguard-tools` |

### Шаг 2 — установите Wren

1. Откройте <https://github.com/j0ck4/wren/releases> и
   скачайте последний файл `wren-vX.Y.Z.flatpak`.
2. **Дважды кликните** по файлу. Откроется GNOME Software /
   KDE Discover / аналог — нажмите **Установить**.
3. Найдите **Wren** в меню приложений и запустите.

Всё. При первой установке скачается GNOME runtime (~600 МБ,
один раз — последующие Flatpak-приложения его переиспользуют).

> Если двойной клик ничего не делает, в дистрибутиве нет
> sideload-помощника. Установите из терминала:
> ```
> flatpak install --user ~/Downloads/wren-vX.Y.Z.flatpak
> ```

## Как пользоваться

### Первый запуск

Пустое окно с одной кнопкой **Import .conf**. Файл конфигурации
WireGuard (`.conf`) выдаёт VPN-провайдер или системный
администратор.

### Импорт туннеля

Нажмите **Import .conf**, выберите файл. Туннель появится в
боковой панели. Wren скопирует файл в своё хранилище (с правами
`0600`, чтобы приватный ключ оставался приватным) и автоматически
переименует, если имя длиннее 15-символьного лимита WireGuard.

### Подключение / отключение

1. Кликните туннель в боковой панели.
2. Нажмите синюю кнопку **Connect**.
3. Введите пароль аккаунта когда спросит polkit.
4. Кнопка станет красной **Disconnect**, тост подтвердит, а в
   углу появится системное уведомление.

### Detail-страница

При выборе туннеля справа отображается:

- **Transfer** — счётчики Received / Sent в KiB / MiB / GiB,
  обновляются каждые 2 секунды. Видны только пока туннель
  активен.
- **Interface** — Address, DNS, Listen Port, MTU.
- **Peers** — раскрывающийся ряд на каждый peer с публичным
  ключом, allowed IPs, endpoint, keepalive. ПКМ ▸ Copy работает
  на любом значении.

### Редактирование

Иконка **карандаш** в шапке. Откроется диалог с редактируемыми
полями Interface и списком peers; иконка корзины удаляет peer,
кнопка **+** в шапке секции добавляет новый. **Save** записывает
изменения; **Cancel** отменяет.

### Удаление туннеля

Красная иконка **корзины** в шапке. Появится диалог подтверждения;
если туннель сейчас подключён, он сначала отключается, затем его
`.conf` безвозвратно удаляется из хранилища Wren.

### Поделиться через QR

Иконка **QR**. Откроется диалог с туннелем в виде QR-кода —
отсканируйте мобильным приложением WireGuard
(*Add tunnel ▸ Create from QR code*). Или нажмите
**Copy Configuration**, чтобы скопировать конфиг в буфер.

### Системный трей

Wren регистрирует трей-иконку там, где окружение это поддерживает
(KDE / XFCE / Cinnamon / MATE — из коробки; GNOME — поставьте
[расширение AppIndicator](https://extensions.gnome.org/extension/615/appindicator-support/)).

ПКМ по иконке: **Show Wren**, быстрый список туннелей с
переключателем, **Quit**.

### Автозапуск при логине

**☰ меню** в шапке боковой панели → **Start at Login**.
Снять галочку — выключит автозапуск.

### Закрытие окна с активными туннелями

Если закрыть окно при активных туннелях, появится диалог:

- **Cancel** — оставить окно открытым.
- **Disconnect & Quit** — опустить всё и закрыть.
- **Quit Anyway** — закрыть окно; туннели продолжат работать
  на хосте.

### Как часто запрашивается пароль

При первом подключении в admin-сессии — да, спросит пароль.
Polkit-policy (`auth_admin_keep`) кэширует аутентификацию на ~5
минут, так что следующие connect/disconnect в этом окне пароль
не запрашивают.

> Нюанс: внутри Flatpak-песочницы файл политики не
> устанавливается системно, поэтому промт может появляться
> каждый раз. Native установка через `apt` решает это и
> запланирована для Ubuntu 26.04+.

## Решение проблем

**`wg-quick: command not found` при подключении**
Пропустили Шаг 1 установки — `sudo apt install wireguard-tools`.

**`config file must be a valid interface name, followed by .conf`**
В имени файла странные символы или оно длиннее 15 знаков. Wren
должен переименовать автоматически; если нет — переименуйте сами
в `~/.config/wren/tunnels/`.

**Иконка трея не появляется на GNOME**
Поставьте AppIndicator-расширение (ссылка выше), выйдите из
системы, зайдите снова.

**`Could not refresh tunnel status`**
В дистрибутиве нет `iproute2` (крайне редко). Установите.

**Туннель поднят, но нет интернета**
Это проблема *конфига* WireGuard, не Wren. Проверьте, что
сервер доступен, ключи совпадают с серверными, а `AllowedIPs`
адекватные. `sudo wg show` покажет состояние со стороны ядра.

**Пароль запрашивается на каждый клик**
Вы на Flatpak-версии — это ожидаемо (см. *Как часто
запрашивается пароль* выше).

## Где лежат файлы

Конфигурации туннелей хранятся здесь:

- Flatpak: `~/.var/app/io.github.j0ck4.Wren.Devel/config/wren/tunnels/`

Можно положить `.conf` туда напрямую, если файловый менеджер
удобнее диалога импорта.

## Сообщить о баге

Откройте issue: <https://github.com/j0ck4/wren/issues>.

К багу полезно приложить:

- Дистрибутив и DE (`uname -a`, `echo $XDG_CURRENT_DESKTOP`)
- `.conf` который пытались импортировать — **затрите PrivateKey!**
- Логи: запустите из терминала
  `RUST_LOG=wren=debug flatpak run io.github.j0ck4.Wren.Devel`
  и приложите всё, что выглядит относящимся
